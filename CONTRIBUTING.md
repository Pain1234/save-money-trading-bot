# Contributing

Thank you for interest in the **public core** of this project.

## Scope

Contributions should improve the public research / paper / infrastructure framework.
Do **not** open PRs that add:

- private strategy candidates, optimized parameters, or confidential results
- live trading enablement or exchange credentials
- production account data or private telemetry

See [docs/governance/PUBLIC_PRIVATE_BOUNDARY.md](docs/governance/PUBLIC_PRIVATE_BOUNDARY.md).

## Process

1. Open or reference a **GitHub issue** first (one issue → one branch → one PR).
2. Follow AGENTS.md and docs/DEFINITION_OF_DONE.md.
3. Keep PRs focused; include the project PR template sections (risks, rollback, research impact, security).
4. CI must be green before merge.

## Conduct

Be respectful. Do not submit secrets. Do not claim profitability or give financial advice in docs or issues.
